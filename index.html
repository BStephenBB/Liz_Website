Dashboard Widgets - Layout

This is a before screenshot of the existing Electric Flow DevOps Insights Dashboard. There were many issues because there weren’t many UX implementations designed to make dashboarding a user-friendly process. In addition to coming up with various use case solutions for displaying time through filtering, I was given the task of going through all the specific widgets and coming up with new widget solutions that would affect both the visual and usability elements of the widgets. The layout of the dashboards changed from a maximum of 8 widgets to 6, so the widgets wouldn’t have readability issues at the minimum window size.

Skills used: Sketch

Dashboard Widgets - Stacked Area Graph

This new widget solution affected the Area Graph, Stacked Area Graph, Line Graph, and Multi-series Line Graph because they all consisted of the same widget layout and functionality. Vertical lines were added so the user would make the most of the x-axis data and labels. There was also more structure to the legend, and the padding gives room for a double-stacked legend.

Skills used: Sketch

Dashboard Widgets - Stacked Area Graph

In the old widget, users must click and drag horizontally on the graph area to view the data, and when they let go of the mouse, they were taken directly to the data center to view the selected dates. This means there’s no indication to the user that a click and drag functionality exists since it only pertains to the Area and Line graphs. The second problem is the user is involuntarily brought to the data center, and cannot edit their selection. My solution solves both issues. First I added a hover instruction for the user explaining how to interact with the drill-down feature. Second, I gave users the ability to edit and cancel their selection if they accidentally clicked and dragged.

Skills used: Sketch

Dashboard Widgets - Donut Graph

This is the only widget that had a clickable legend because the users weren’t able to select the slices when they were too small. I removed the white spacing between the slices since they often appeared thicker than the slice itself, causing the users to think they were slices. I also added a magnifying element on hover so users could enlarge the data. It is vital for the users to see the percentages without hover because the dashboard feature is often displayed on a projector, and users must be able to get as much information at a glance. The old version only showed percentages around the donut graph if the percentage was larger than 20% because dev couldn’t squeeze in the data elegantly with the thinner slices. In my new solution, the legend turned vertical, displaying all percentages in a breathable way.

Skills used: Sketch

Dashboard Widgets - Stacked Bar Graph

The issues with the bar graphs and (not pictured) histograms was the use of white space and the inability to see the thinner bars, which was an existing problem in the donut graph that was not caught for the bar graph.

Skills used: Sketch

Dashboard Widgets - Table Chart

The table had unaesthetic behavior and few layout rules. The white space wasn’t used efficiently and the table elements were clickable, but weren’t responsive on hover. The new version had an underline on hover to indicate clickability. I also added Column Labels so users know what the data means, as the widget titles are variable.

Skills used: Sketch

Dashboard Slideshow - Dashboard

The issue with the previous version of the DevOps Insights Dashboards was that users couldn’t enlarge the widgets. My solution was to add expand buttons to the upper right corner of each widget.

Skills used: Sketch

Dashboard Slideshow - Slideshow

I took the expanded view to another level by introducing the slideshow view allowing users to remain in the expanded states and simultaneously view the other widgets via side arrow navigation or bottom pagination dots. The pagination dots also include a hover feature so users can navigate to the specific widget they want to view.

Skills used: Sketch

Heat Map - Color Palette

I was tasked with figuring out a color scheme that would indicate High, Medium, Low, and N/A for users looking at the data at a fast glance. I chose contrasting monochromatic colors to make the data more accessible to color-blind users. I played around with different text colors, going from all black texts to all white texts to mixed colors, and tested the readability on multiple projectors around the office.

Skills used: Sketch, Illustrator

FitFifteen - Moodboard

FitFifteen stemmed from a design final and turned into a quarter-long special study project under a UX design professor. It is a fitness app centered around social change seeks to bridge the gap between the fitness industry and everyone else. It targets non-gym goers and busy students and people with time-consuming jobs. This is a mood board that I made for the project from either existing workout apps, or apps with a clean design that I wanted to get inspiration from.

Skills used: Sketch, Illustrator, HTML5, CSS, Javascript, Version Control, Firebase
